# Google Drive 선택 저장 설정

## 적용 위치

이번 기능은 `💰 매출계산` 페이지가 아니라 **`📥 엑셀 업로드` 페이지**에서 실행됩니다.

엑셀 업로드 페이지에 아래 라디오 버튼이 표시됩니다.

```text
업로드한 엑셀을 Google Drive에 저장할까요?
○ 예   ○ 아니오
```

동작 기준은 다음과 같습니다.

- `예`를 직접 선택한 경우에만 Google Drive에 저장합니다.
- `아니오`를 선택하면 저장하지 않습니다.
- 아무것도 선택하지 않은 기본 상태에서는 저장하지 않습니다.
- Drive 저장 실패가 발생해도 기존 엑셀 복호화/집계/PDF/TC 엑셀 생성 기능은 멈추지 않고 계속 실행됩니다.

## 저장 순서

사용자가 `예`를 선택한 상태에서 엑셀을 업로드하면 아래 순서로 동작합니다.

1. 업로드한 원본 엑셀 파일 bytes 확보
2. Google Drive `판매내역` 폴더 찾기
3. 오늘 날짜 폴더 찾기 또는 생성
   - 예: 4월 24일이면 `4.24`
4. 같은 이름의 파일이 있으면 기존 파일을 최신 업로드본으로 덮어쓰기
5. 같은 이름의 파일이 없으면 새로 저장
6. 기존 엑셀 복호화/집계/PDF/TC 엑셀 생성 기능은 그대로 계속 실행

## Render 환경변수

필수:

```text
GOOGLE_SERVICE_ACCOUNT_JSON={서비스계정 JSON 전체}
GOOGLE_DRIVE_SALES_ROOT_FOLDER_NAME=판매내역
```

권장:

```text
GOOGLE_DRIVE_SALES_ROOT_FOLDER_ID=판매내역 폴더 ID
```

`GOOGLE_DRIVE_SALES_ROOT_FOLDER_ID`를 넣으면 같은 이름의 다른 폴더가 있어도 정확히 그 폴더 안에 저장됩니다.

## Google Drive 권한

Google Drive의 `판매내역` 폴더를 서비스 계정 이메일에 **편집자 권한**으로 공유해야 합니다.

서비스 계정 이메일은 `GOOGLE_SERVICE_ACCOUNT_JSON` 안의 `client_email` 값입니다.

## 저장 예시

오늘이 4월 24일이고, 엑셀 업로드 페이지에서 `네이버매출1.xlsx`를 업로드하면:

```text
판매내역 / 4.24 / 네이버매출1.xlsx
```

이미 같은 파일명이 있으면 새 파일을 하나 더 만들지 않고 기존 파일 내용을 최신 파일로 업데이트합니다.
